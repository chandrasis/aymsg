yahoo64

public static final String yahoo64(byte[] buffer)
Yahoo uses its own custom variation on Base64 encoding (although a little birdy tells me this routine actually comes from the Apple Mac?) For those not familiar with Base64 etc, all this does is treat an array of bytes as a bit stream, sectioning the stream up into six bit slices, which can be represented by the 64 characters in the 'table' Y64 above. In this fashion raw binary data can be expressed as valid 7 bit printable ASCII - although the size of the data will expand by 25% - three bytes (24 bits) taking up four ASCII characters. Now obviously the bit stream will terminate mid way through an ASCII character if the input array size isn't evenly divisible by 3. To flag this, either one or two dashes are appended to the output. A single dash if we're two over, and two dashes if we're only one over. (No dashes are appended if the input size evenly divides by 3.)
